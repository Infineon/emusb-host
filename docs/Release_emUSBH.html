Release notes for emUSB-Host V2.48.0

Version 2.48.0 [12. Aug 2025]

New Features

  1. Video class: Added new functions USBH_VIDEO_GetStreamState() and USBH_VIDEO_RestartStream() which allow dealing with packet level errors from error prone webcam devices.
  2. FT232: Support baud rates up to 12MBaud for high-speed devices.
  3. HID: Add function USBH_HID_SuspendResume().

Program corrections

  1. Video: In rare cases when the underlying ISO transfer experienced a transfer error it was possible for the user data callback to not be called. Fixed.

Version 2.46.0 [26. Jun 2025]

New Features

  1. Support USB test mode for LPC55xxx devices.
  2. Support Microship SAMD51 devices.
  3. Add Support for WCH CH34X USB to RS232 converter.

Program corrections

  1. USBH_FT232_Exit could cause NULL pointer de-reference when USBH_FT232_AddCustomDeviceMask() is not used. Fixed.
  2. Using the function USBH_FT232_Read with the STM32_NG driver could cause hardfault by interpreting data as valid. Fixed.

Version 2.44.0 [25. Feb 2025]

New Features

  1. Add support for STM32H7Sxx devices.

Improvements

  1. DWC2 (Synopsys) DMA: Ping protocol enabled.

Version 2.42.1 [12. Aug 2024]

Improvements

  1. EHCI driver (iMX6,iMXRT10xx/11xx): Remove 250ms delay for HS disconnect detector enable.

Version 2.42.0 [26. Jun 2024]

Improvements

  1. DWC2 (Synopsys): Improve critical section locking on transfer aborts.

Program corrections

  1. AUDIO: For devices with multiple streaming interfaces, some interfaces may not be detected. Fixed.
  2. General: Reading a string descriptor did return illegal data if a timeout occurred during the control request. Fixed.
  3. DWC2 (Synopsys) DMA: Missing initialization of the 'Aborted' flag may cause unwanted transfer aborts after recovery of stuck channels. Fixed.
  4. DWC2 (Synopsys) DMA: High bandwidth ISO OUT transfers did not work correctly. Fixed.

Version 2.40.0 [19. Mar 2024]

Improvements

  1. AUDIO: Changed API for easier use.
  2. PSoC6 driver: Use plugin for DMA routines to enable support for different DMA controllers.

Version 2.36.7 [22. Feb 2024]

Improvements

  1. LAN: IP config for emNet was updated to recognized newer RealTek 1G/2.5G ethernet adapter with more than 2 configuration descriptors.

Program corrections

  1. LAN: Updated the ASIX 179x adapter initialization. On some devices the LED configuration may fail. Fixed
  2. LAN: ASIX PHY adaptation for emNet may call the ASIX178/179 reset initialization multiple times even when only one adapater is available. Fixed
  3. LAN: ASIX RX callback for ASIX 179 devices did not check for busy status which may result in stopping RX transfers. Fixed

Version 2.36.6 [23. Oct 2023]

Program corrections

  1. DWC2 (Synopsys) full-speed driver: USB controller was not initialized correctly and may not work. This bug was introduced in version 2.36.5. Fixed.

Version 2.36.5 [14. Oct 2023]

New Features

  1. PSoC6 driver: Support DMA for packet transfers.
  2. OTG: Added driver for XMC4500 devices.

Program corrections

  1. STM32H5xx / STM32G0xx driver: Avoid race condition on USB control registers that may break data transfers.
  2. LPC54/55xx HS driver: Setup request may not work after a stall. Fixed.
  3. Enumeration: If V2.10 features are enabled, the stack sometimes tries to read the BOS descriptor for V2.00 devices. Fixed.

Version 2.36.4 [29. Sep 2023]

New Features

  1. LAN module: Added new function: USBH_LAN_SetOnAddRemove().
  2. DWC2 (Synopsys) DMA driver: Allow high bandwidth ISO transfers.

Improvements

  1. DWC2 (Synopsys) driver: Speed up controller recovery, if required after disconnecting device from root port.

Program corrections

  1. Increased robustness of the parser against malformed configuration descriptors that may be provided by broken devices. The issue was reported by the MCUSec lab at the University of Georgia.
  2. LAN module: When a device has been removed it was possible for the async completion callback to access the already freed device instance, which could result in a HardFault. Fixed.

Version 2.36.3 [21. Jun 2023]

New Features

  1. Renesas driver: Add new function USBH_ConfigOvercurrentDetection().

Improvements

  1. PSoC6 driver: Use USB controller reset at start-up and on fatal errors.
  2. HID class: Improved management of HID device instances.
  3. CDC class: Improve management of CDC device instances.
  4. General: Added string descriptor indexes to the USBH_INTERFACE_INFO structure for easy access using USBH_GetStringDescriptor.

Program corrections

  1. HID class: The memory structure of malfunctioning HID devices may not have been removed from the USB stack after disconnection in some cases. Fixed.

Version 2.36.2 [16. Jan 2023]

Improvements

  1. OHCI: Update driver to enable use of URB queue.
  2. Legacy MSD module removed.

Version 2.36.1 [15. Nov 2022]

New Features

  1. PSoc6 driver: New macro to configure interrupt level.
  2. Added BSP and configuration files for the ST STM32H735G-DK evaluation board.

Improvements

  1. CDC: Removed unnecessary DEC_REF_CNT, which caused a warning with USBH_REF_TRACE enabled.

Version 2.36.0 [28. Jul 2022]

New Features

  1. Added Video class.

Improvements

  1. PSoc6 driver: Added USB line (D+/D-) configuration in driver.

Program corrections

  1. EHCI driver: Control requests with large data sizes did not work, when small transfer buffer sizes are configured. Fixed.
  2. EHCI driver: Isochronous high-bandwidth transfers could contain garbage data under certain circumstances. Fixed.

Version 2.34.0 [13. Apr 2022]

New Features

  1. CP210x: Support for CP2013 and CP2014 added.
  2. HUB: Configure hubs to operate as a multiple TT organized hub (if supported).
  3. Added support for Cypress PSoC 6 devices.

Program corrections

  1. STM32H7xx driver: FIFO buffer configuration caused malformed interrupt out split transactions. Fixed.
  2. DWC2 (Synopsys) HS driver: Aborting a split transaction due to a timeout was not performed correctly. Fixed.
  3. FT232: USBH_FT232_GetQueueStatus was not implemented correctly. USBH_FT232_Read would discard any data which was not read by the application immediately (was saved into the internal buffer). Fixed.
  4. CP210x: Made sure USBH_CP210X_Close only sends the "Disable" command once.

Version 2.32.1 [03. Dec 2021]

New Features

  1. Added support for Renesas RA6M3 devices.

Program corrections

  1. Printer/FT232: Both classes only allowed 32 enumerations in total each. Fixed.
  2. LAN: Using the LAN module with emNet version >= V3.42 did not compile. Fixed.
  3. Renesas HC: Control-Out request with MaxPacketSize or higher did not work properly. Fixed.
  4. Renesas HC: Under some circumstances USB FIFO read transfers from Renesas RX controller did not read correct data. Fixed.

Version 2.32.0 [24. Sep 2021]

New Features

  1. Added function USBH_SetOnPortEvent() to report port events to the application.
  2. Added support for Renesas RA6M4 devices.

Version 2.30.1 [28. Jul 2021]

New Features

  1. Added support for Giga Device GD32F450 devices (full-speed only).

Improvements

  1. FT260: Fix issues for older compilers.

Version 2.30.0 [14. Jul 2021]

New Features

  1. HID: Added support for feature reports.
  2. FT260: Support for FTDI FT260 chips.
  3. embOS layer: Support system tick rates different from 1 ms.
  4. FreeRTOS layer: Support system tick rates different from 1 ms.

Program corrections

  1. If the option USBH_DELAY_BETWEEN_ENUMERATIONS is used, a USB port may become permanently blocked after an enumeration error. Fixed.
  2. CDC-ECM: The mac address of a CDC-ECM device could not retrieved when no LANGID in the GetStringDescriptor was used. Fixed.
  3. Renesas RX drivers: Maximum number of devices that can be handled simultaneously was wrong. Fixed.

Version 2.28.0 [09. Apr 2021]

New Features

  1. Added function USBH_GetIADInfo().
  2. Added support for iMXRT1170 devices.
  3. Added new driver - MUSB with support for TM4C1294.
  4. HID: Added plugin for remote control devices (remote control HIDs are often part of USB audio devices with buttons for volume and track control).
  5. OHCI: Added zero-copy variant for the driver. This variant of the OHCI driver will try to use zero-copy where possible, thereby increasing speed and reducing memory consumption.

Improvements

  1. General: Reworked logging functionality, added function USBH_ConfigMsgFilter().
  2. OHCI: Reduced overall memory consumption.
  3. MSC: Added a workaround for certain USB sticks which can under certain circumstances erroneously send a zero-length-packet in the data phase which would cause the sticks to not be recognized.
  4. ASIX: Added support for gigabit Ethernet adapters (chipsets AX88178, AX88178a, AX88179).

Program corrections

  1. LPC55S69: Devices would enumerate only once when connected to the root-hub port. Fixed.
  2. EHCI driver: Isochronous transfers did not work with full-speed devices connected to the root hub. Fixed.
  3. EHCI driver: Isochronous high-speed transfers with interval less than 1ms were sometimes garbled. Fixed.
  4. CDC: Function USBH_CDC_SuspendResume() has disconnected the device, so it can't be resumed. Fixed.
  5. Printer: Function USBH_PRINTER_GetDeviceId() did not zero-terminate the returned string. Fixed.

Version 2.26.1 [15. Dec 2020]

New Features

  1. Bulk: Added function USBH_BULK_GetMaxTransferSize()
  2. Bulk: Added function USBH_CDC_GetMaxTransferSize()
  3. Printer: Added function USBH_PRINTER_AddCustomDeviceMask()

Improvements

  1. HID: Divided HID into a main parser and different plug-able client modules for keyboard, mouse and touch-pad. API remains unchanged, this improvement allows for easy creation of new HID client plugins.

Version 2.26.0 [21. Nov 2020]

New Features

  1. Added support for Renesas RX72N controllers.

Improvements

  1. MSD module refactored: Smaller memory footprint, more tolerant against memory sticks that are not fully compliant to the specification. Enumeration of slow or buggy devices does not delay the whole USB stack any more. New MSD module is used, if USBH_USE_LEGACY_MSD is set to 0.
  2. Core stack: Use more portable timestamp handling.

Program corrections

  1. DWC2 (Synopsys) driver: Access to freed memory when calling USBH_Exit(). Fixed.
  2. Legacy MSD module: Remove ClrFeature on EP 0, removed retries on GetMaxLun, Fixed _AllocLuns getting 0 as MaxLUN, removed handling of "residue" as it is often incorrect in USB sticks.
  3. CDC + FT232: On devices with multiple CDC/FT232 instances sometimes the data interfaces were assigned to the wrong control interface. Fixed.
  4. LAN RNDIS/ECM: Fixed an issue where communication did not work because OnIntStateChange was not set again after USBH_CDC_Close. (Previously USBH_CDC_Close behaved differently).

Version 2.24.7 [29. May 2020]

Improvements

  1. Enumeration: Improved state machine behaviour for quick disconnections during an enumeration.

Program corrections

  1. MSD: USBH_MSD_GetUnits() returned USBH_STATUS_SUCCESS for DevIndex parameters which were not valid. Fixed.
  2. MSD: When using the Renesas or the DWC2 (Synopsys) FS driver sometimes a MSD device's reference counters could become negative, the stack would stop in USBH_PANIC in the debug build in such a case. The issue was introduced in 2.24.6. Fixed.
  3. MSD: It was possible for the SetupURB to be used twice when a setup transaction resulted in a call to _UsbDeviceReset, which in turn could result in a device not being freed after removal. Fixed.
  4. Renesas HS driver: In rare cases low-speed USB devices did not enumerate. Fixed.

Version 2.24.6 [17. Mar 2020]

Program corrections

  1. MSD: Some memory structures may not be freed if removing a memory stick. The issue was introduced in 2.24.4. Fixed.

Program corrections

Version 2.24.4 [17. Jan 2020]

Improvements

  1. Printer class: Added functions USBH_PRINTER_Receive() and USBH_PRINTER_WriteEx().
  2. MSD: The MSD module no longer calls FS_UnmountForced() internally if an error during a write or read operation occurred and the recovery was not successful.

Program corrections

  1. Printer class: Previously printers lacking the optional bulk IN endpoint were not recognized by the printer class. Fixed.
  2. Printer class: USBH_PRINTER_GetDeviceId(), USBH_PRINTER_ExecSoftReset() and USBH_PRINTER_GetPortStatus() could fail when the device's printer interface number was not the same as the emUSB-Host internal interface ID. Fixed.
  3. MSD: In very rare cases if an error occur during the exchange of MSD initialization commands it was possible for the MSD module to get stuck. Fixed.
  4. MSD: In very rare cases if an error occur during an InquiryDevice command inside the initialization phase the MSD module could access invalid memory. Fixed.

Version 2.02 [30. Nov 2017]

Improvements

  1. MSD: Module refactored. Multiple buffers and unnecessary alloc/free operations removed. TestUnitReady command is now sent alongside of data transmissions instead of via a timer. This slightly improves performance.
  2. MSD: Removed BulkOnlyErrorRecovery support, this feature only caused problems as it re-enumerated the device and expected the application to work with the old handle. This has been replaced by a simple reset, when an MSC device misbehaves it is reset and the user receives a removal and an add notification.
  3. MSD: Changed module to allow devices without storage media to be added successfully (e.g. empty SD card reader). Media can be connected to the device at a later time.
  4. Printer: Module refactored. Adding/removing devices to the module is now handled similarly to the other class modules, via reference counters and timers.

Program corrections

  1. USBH_Exit() may hang if called during port reset. Fixed.
  2. STM32F4/F7 (HS) driver: Setup request may not executed correctly if buffer was not aligned. Fixed.
  3. STM32F4/F7 (HS) driver: USB host controller reset sequence may not work when internal Full-Speed PHY is used. Fixed.
  4. Renesas driver: Interval calculation for interrupt endpoint corrected.
  5. Renesas driver: Setup requests and FIFO access did not work on big endian CPUs. Fixed.
  6. Renesas driver: During parallel transactions interrupts may not be handled correctly. Fixed.
  7. Renesas driver: In certain circumstances a FIFO may get confused if it contained an odd number of bytes. Fixed.
  8. Renesas driver: Data toggle may sometimes not processed correctly. Fixed.
  9. HUB support: Stacking multiple HUBs may lead to problems on disconnect. Fixed.
  10. HUB support: In certain circumstances polling for port status changes was stopped. Fixed.
  11. MSD: USBH_MSD_GetUnits could return an incorrect number of units. Fixed.
  12. MSD: During the data phase a zero length packet would cause the module to wait for the read timeout and would consider it to be an error, which is not correct in some cases. Fixed.
  13. KinetisFS: Check for short packets during the setup data phase was invalid, this would cause devices with any requested descriptor longer than the device's endpoint zero MaxPacketSize to fail enumeration. Fixed.
  14. KinetisFS: On EP0 transactions the driver changed the return status from any error to "not responding". Fixed.
  15. OHCI: When an URB was aborted from inside a completion callback (only happens when a hub is removed with attached devices) the TD lists were modified, this could happen while the abort URB timer was running, which would consequently cause already freed TDs to be used, this in turn could, in rare cases cause a magic assert in the debug build or unpredictable behaviour in the release build. Fixed.

Version 2.00b [17. Oct 2017]

Improvements

  1. OHCI: Improved control and bulk EP removal. Previously endpoint lists would be halted for EP removal which could block a new URB from executing for a short time. In cases where an URB is started from inside a timer routine (like in the MSD class) the URB would always timeout because the endpoint list was disabled and could not be re-enable as it was also done from inside a timer. With this version endpoint lists are no longer halted during removal.

Program corrections

  1. Keil RTX/CMSIS RTX v1 RTOS layers: Removed task lock/unlock from USBH_OS_SignalNetEvent. This could cause send/receive functions to fail under rare circumstances. Fixed.
  2. EHCI: USB Interrupts were not disabled on USBH_Exit(), this could lead to a panic if a new device was connected while emUSB-Host was shut down. Fixed.
  3. Synopsys: USB host controller reset sequence did not work for all MCUs of the STM32F4 family. Fixed.

Version 2.00a [21. Sep 2017]

Program corrections

  1. MSD: 50ms timeout introduced in V2.00 caused communication problems with some USB sticks. Increased timeout for the TestUnitReady command back to 10s because some sticks seem to need this after data has been written onto them. Fixed.

Version 2.00 [14. Sep 2017]

Hole USB-Host stack was refactored. History has been reset. Older release notes can be found in older versions of this document.
